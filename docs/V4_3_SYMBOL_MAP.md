## Caveats

- Some 'aliases' may not be aliases at all. Example: `asset_10_8000`, `palette_07_8000`, `lz_compress_03_8000` all end in `_8000` but refer to *different bank/asset slots* at offset `$8000` in different banks (the leading number is a bank/index, not part of the addr). Do **not** blindly merge these.
- Addresses with a leading-zero variant (e.g., `008E88` vs `8E88`, `049000` vs `49000`) are almost certainly the same routine but disassembled with different long-pointer formats; safe to merge after spot-check.
- Pairs that differ only by an underscore/typo (e.g., `cooperative_yield_877D` vs `coop_yield_877D`, `fade_in_85FC` vs `inidisp_on_fade_85FC`) are strong merge candidates.
- Semantic conflicts (e.g., `spider_eats_ant_C0FD` vs `ant_lion_tick_C0FD`) indicate that two reverse-engineering passes disagreed about what the routine *does*. Pick the more accurate name after reading the body, then merge.
- This map only flags symbols whose name ends in a hex-looking suffix. Pure semantic duplicates with no embedded address are NOT covered.
