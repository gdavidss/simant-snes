# SimAnt SNES decomp — Coverage Analysis (V3-C)

_Generated by `coverage_analysis.py` against ROM `simant.sfc` (1 MB LoROM).  Reflects state of `simant-re/` at run time._

## Methodology

1. Each lifted function name encodes a ROM address: `sub_XXXX` (bank-0), `sub_BXXXX` (5-char: bank B, off XXXX), or `sub_BBXXXX` (6-char: explicit bank).
2. For every lifted function we walk the ROM at its entry until the first `RTS/RTL/RTI/JMP/JML/BRA` terminator (capped at 2 KB) to estimate its size, and mark those bytes covered.
3. Code-vs-data per 256-byte chunk: a chunk is **code** if it contains a JSR/JSL target, has ≥4 `RTS/RTL/RTI` opcodes, or is >50% already covered. Otherwise it is **data**.
4. Call counts: every `name(...)` in any function body (across all .c files) counts as one call site for that name.
5. Stub detection: a function whose body is empty or `return 0;`; or has the `__attribute__((weak))` linkage.
6. Call targets discovered by linearly disassembling every bank and collecting `JSR/JSL/JMP/JML` operands.
