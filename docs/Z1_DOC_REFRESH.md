## Contradictions not resolvable from existing docs alone

- **Exact stub count.** AUDIT_SUMMARY now says "~8 entity stubs"
  based on the spec's count (32 + 78 = ~110, leaving ~8). A precise
  count would need a grep over the current `entities_*.c` for
  empty-bodied dispatchers; left as "~8" per spec.
- **Exact bug-fix count.** Spec says ~55, actual could be ~80+ if all
  G2/G3/G4/H1 dispatcher fixes are counted individually. Z1 uses
  ~80 across the README/COVERAGE/AUDIT_SUMMARY to bracket the range.
- **Total line count.** `wc -l *.c` on the current tree reports
  68,545; A2 / spec says 68,517. Difference of 28 lines is likely
  trailing-newline drift across files added after the audit snapshot.
  Z1 uses the spec's 68,517 to stay consistent with the A2 audit.
- **COVERAGE_ANALYSIS.md per-bank tables.** These were emitted by
  `coverage_analysis.py` against the 46-file snapshot and contain
  derived byte-coverage stats that would require re-running the
  script to refresh. Z1 only corrected the headline file/line
  numbers and added a callout; the script re-run is left as a
  separate task.
